# Базы данных из IDE (kinescope-dev-mcp)

MCP-сервер **kinescope-dev-mcp** подключает ассистента к **PostgreSQL** и **ClickHouse**. Вы не переключаетесь в отдельный клиент — просто просите в чате проверить схему, выполнить запрос или посмотреть здоровье БД.

## Что можно делать и как это выглядит

| Что хотите | Что пишете в чат |
|---|---|
| Посмотреть структуру таблицы | «покажи структуру таблицы botdetect.ip_blacklist» |
| Выполнить SQL | «выполни этот запрос на ClickHouse: SELECT ...» |
| Проверить схему перед миграцией | «какие колонки и индексы в таблице botdetect.events?» |
| Диагностика PostgreSQL | «проверь здоровье БД» |
| Найти проблему | «pq: permission denied for schema botdetect» (вставляете ошибку — ассистент сам проверяет коннекты и права) |

Cursor сам разбирается, к какой БД обратиться и в каком порядке получить информацию. Вам не нужно знать, какие инструменты есть у MCP-сервера.

## Практические сценарии

### Проверить схему перед написанием запроса или кода

```text
Покажи структуру таблицы botdetect.events в ClickHouse — какие поля, типы, партиционирование
```

Cursor покажет колонки, типы, движок и ключи. Так реже случаются опечатки в именах полей и несоответствия типов.

### Аналитика и отладка по событиям

```text
Выполни на ClickHouse: SELECT ip, count() FROM botdetect.events WHERE timestamp > now() - INTERVAL 1 HOUR GROUP BY ip ORDER BY count() DESC LIMIT 20
```

Для потоков событий в ClickHouse полезно явно ограничивать время и объём выборки. Используйте это для проверки гипотез о трафике, аномалиях или регрессиях после выката.

### Здоровье БД и индексы (PostgreSQL)

```text
Проверь здоровье PostgreSQL — есть ли проблемы с индексами или раздутием таблиц?
```

Помогает увидеть узкие места до того, как они станут инцидентом, и понять, нужен ли новый индекс или стоит удалить неиспользуемые.

### Миграции с опорой на текущую схему

Перед добавлением столбца или таблицы:

```text
Покажи текущую схему таблицы botdetect.ip_blacklist — колонки, типы, ограничения, индексы
```

Так миграция не конфликтует с существующими ограничениями. После наката на стенд можно проверить результат тем же способом.

### Разбор ошибок

Реальный пример: в логах появилось `pq: permission denied for schema botdetect`. Вставляете ошибку в чат — ассистент сам проверяет коннекты к PostgreSQL, находит, что отсутствует `GRANT USAGE ON SCHEMA`, и предлагает миграцию.

## Безопасность

- **По умолчанию — чтение**: SELECT, просмотр метаданных, диагностика.
- **Запись и DDL** — только после явного подтверждения.

Не полагайтесь на ассистента как на замену code review для миграций: он ускоряет исследование схемы, но ответственность за прод остаётся на команде.

## Связка с остальным стеком

Запрос к БД часто идёт в одной цепочке с Kaiten (критерии задачи) и Пачкой (обсуждение инцидента). MCP позволяет не рвать этот контекст: карточка, переписка и фактическая схема данных доступны из одной сессии в IDE.
