# Инфраструктура из IDE (kinescope-ops-loc)

MCP-сервер **kinescope-ops-loc** подключает ассистента к Proxmox-серверам. Не нужно логиниться в веб-интерфейс PVE — просто просите в чате.

## Что можно делать и как это выглядит

| Что хотите | Что пишете в чат |
|---|---|
| Список всех VM | «покажи все QEMU VM по всем Proxmox-серверам» |
| Информация по хостам | «покажи список хостов с CPU/RAM/Disk» |
| Детали конкретной VM | «покажи подробную информацию по VM Kirill» |
| Storage на хосте | «покажи storages на ru-msk-ost-infra-pve-2» |
| Где мало ресурсов | «подсвети хосты где ресурсов мало» |
| Группировка данных | «сделай таблицу с группировкой по server и status» |

Cursor сам разбирается, какой инструмент вызвать и как агрегировать результат. Вам не нужно знать API Proxmox.

## Практические сценарии

### Инвентаризация VM

`pve_list_vms` возвращает все VM со всех PVE-серверов: dc, server, node, vmid, name, status, cpu, ram, disk. Cursor выводит таблицу и может агрегировать: по серверу, по статусу, суммарные ресурсы.

### Capacity planning

`pve_list_hosts` возвращает хосты с CPU/RAM/Disk + storages + vCPU assigned. Cursor может применить пороги и подсветить проблемы: RAM < 10%, storage < 100G, vCPU overcommit.

### Детальный drill-down

От общей картины к конкретным VM на проблемных хостах. «покажи VM на машинах где мало места» — Cursor запросит хосты, отфильтрует по порогу, затем покажет VM с привязкой к storage.

### Troubleshooting MCP

Если PVE-сервер не отвечает (timeout, 401), Cursor покажет ошибку в отдельной колонке. Можно повторить запрос после фикса конфига или сети. Итеративный процесс: timeout -> auth fix -> URL fix -> success.

## Агрегация данных

Cursor может в одном ответе: запросить данные, отфильтровать, агрегировать, применить пороги, вывести таблицу с алертами. Не нужно писать скрипты — формулируйте на естественном языке.

## Сохранение отчётов

Результат можно сохранить в файл: «сохрани в low_cap.md» — Cursor создаст markdown-файл с таблицами, пригодный для wiki или передачи коллегам.

## Доступные инструменты

| Tool | Что делает |
|---|---|
| pve_list_vms | Список QEMU VM со всех PVE-серверов (vmid, name, status, ресурсы, диски) |
| pve_list_hosts | Хосты/nodes: CPU, RAM, Disk, storages, vCPU assigned, ошибки опроса |

## Безопасность

По умолчанию — чтение. MCP-инструменты для Proxmox в текущей конфигурации read-only: list_vms, list_hosts. Управление VM (старт/стоп/миграция) потребует отдельных инструментов с подтверждением.

## Связка с SSH

Для глубокой диагностики (smartctl, mdadm, lsof) MCP недостаточно — нужен SSH. Но для обзора инфраструктуры, capacity planning и инвентаризации MCP быстрее и удобнее.
