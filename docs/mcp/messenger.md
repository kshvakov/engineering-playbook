# Мессенджер (Пачка) из IDE

Через тот же MCP-сервер **Jarvis** ассистент получает доступ к **Пачке**. Цель — читать обсуждения, треды и каналы из IDE, чтобы ассистент опирался на то, что уже решили в чате, а не только на локальные файлы и карточку задачи.

## Что можно делать и как это выглядит

| Что хотите | Что пишете в чат |
|---|---|
| Найти обсуждения по задаче | «найди обсуждения по задаче 62353324 в Пачке» |
| Прочитать канал | «покажи последние сообщения из канала backend» |
| Прочитать тред | «покажи тред из Пачки по ссылке ...» |
| Собрать контекст | «посмотри карточку в Kaiten и найди связанные обсуждения в Пачке» |

Вы не указываете, какие инструменты вызывать — просто описываете что нужно.

## Сценарий: контекст вокруг задачи

Вы работаете по карточке Kaiten. Попросите ассистента найти обсуждения:

```text
Найди в Пачке обсуждения, связанные с этой задачей
```

В ответе — сообщения, где обсуждали задачу, кидали ссылки, уточняли требования. Так ассистент видит неформальные уточнения, которые редко дублируют в описании тикета.

## Сценарий: требования из канала

Продуктовый или инженерный канал часто содержит договорённости до появления карточки:

```text
Покажи последние сообщения из канала backend за эту неделю
```

По итогам можно сформулировать спецификацию, чеклист или черновик карточки в Kaiten. Полезно явно указать временной диапазон или ключевые слова, если канал шумный.

## Сценарий: решение в треде

Когда важна последовательность «вопрос — уточнение — финальное решение»:

```text
Покажи тред из Пачки, где обсуждали формат событий botdetect
```

Тред даёт связный диалог. Имеет смысл сочетать с карточкой Kaiten: в карточке — формальное ТЗ, в треде — нюансы внедрения.

## Автоизвлечение ссылок

Ссылки из сообщений Пачки автоматически извлекаются и обрабатываются:

- ссылки на **Kaiten** дополняются заголовком и статусом карточки;
- ссылки на GitLab, Figma и другие ресурсы сохраняются для навигации.

Это снижает риск, что ассистент прочитает текст сообщения, но проигнорирует вложенный линк на актуальный MR или макет.

## Ограничения

- Доступ определяется учётной записью, под которой настроен Jarvis; не все каналы могут быть видны.
- История может быть усечена — для длинных обсуждений уточняйте период.
- Конфиденциальные темы подпадают под корпоративные политики; MCP не отменяет правила разграничения доступа.

## Связка с таск-трекером

Оптимальная связка для полного контекста задачи — попросить ассистента одновременно посмотреть карточку в Kaiten и найти связанные обсуждения в Пачке:

```text
Посмотри карточку 62353324 и найди обсуждения по ней в Пачке
```

Так мессенджер перестаёт быть отдельным островом и становится частью контекста в IDE.
