# SSH-диагностика из IDE

Cursor подключается к серверам по SSH напрямую из чата (через Shell tool), даёт доступ к полной диагностике Linux без переключения в терминал и анализирует вывод в контексте разговора.

## Зачем

df, du, lsof, smartctl, mdadm, journalctl, ss, ip — всё это можно выполнять на удалённом сервере прямо из IDE. AI парсит вывод, делает выводы, строит следующую команду на основе результатов. Не нужно держать открытыми десять терминалов с SSH-сессиями.

## Как это работает

1. AI формирует SSH-команду: `ssh -o ConnectTimeout=10 hostname "command"`.
2. Выполняет через Shell tool.
3. Парсит вывод, делает выводы.
4. При необходимости строит следующую команду на основе результатов предыдущей.

Каждый вызов — отдельная SSH-сессия; состояние между командами не сохраняется.

## Подключение

- Работает через `~/.ssh/config` и ssh-agent — как обычный SSH из терминала.
- Для новых хостов: `ssh -o StrictHostKeyChecking=accept-new` (AI делает это автоматически при Host key verification failed).
- Для jump-хостов: `ProxyJump` / bastion из ssh config.
- `ConnectTimeout=10` — чтобы не зависать на недоступных хостах.

## Безопасные команды

AI выполняет без подтверждения — только чтение, никаких изменений состояния.

| Область | Команды |
|---------|---------|
| Диски | `df -hT`, `du -xh --max-depth=1`, `lsblk -o NAME,SIZE,TYPE,MODEL,SERIAL` |
| Процессы | `ps`, `top -bn1`, `pidstat` |
| Deleted files | `lsof +L1` |
| RAID | `cat /proc/mdstat`, `mdadm --detail`, `mdadm --examine` |
| SMART | `smartctl -i`, `smartctl -H`, `smartctl -A` |
| Сеть | `ss -tanp`, `ip -s link`, `ip route` |
| Логи | `journalctl -u service --since "1h ago"`, `dmesg -T \| tail` |
| Память | `free -h`, `cat /proc/meminfo`, `vmstat 1 5` |

## Опасные команды

Требуют явного подтверждения пользователя перед выполнением:

- `kill` / `killall`
- `systemctl restart` / `stop`
- `mount` / `umount`
- `sysctl -w`
- `mdadm --add` / `--remove` / `--fail`
- `rm`, `dd`, `mkfs`
- Любая запись в `/proc`, `/sys`

## Параллельный сбор

Когда нужно проверить несколько серверов, AI запускает SSH-команды в параллельных tool calls — быстрее последовательного опроса. Результаты собираются в единый отчёт, обычно в виде таблицы.

Пример: аудит RAID на kinescope-postgres-3 и ru-msk-ixlrt-bcp-1 — два SSH-вызова одновременно, результаты в одной сводной таблице.

## Адаптация к ошибкам

AI автоматически адаптируется к типичным проблемам:

- **Host key verification failed** — повтор с `accept-new`.
- **Command not found** — альтернативная команда (`nvme list` вместо `smartctl /dev/nvme*`).
- **Permission denied** — `sudo` (если доступен).
- **Нет sd-устройств** — проверка nvme-устройств.

## Формат вывода

AI форматирует сырой вывод SSH-команд в читаемые таблицы, структурированные отчёты, рекомендации. Из `smartctl -i` + `mdadm --detail` на пяти серверах получается одна таблица: модель, serial, размер, SMART-статус, RAID status.

## Ограничения

- **Нет интерактивности:** не работают vi, less, htop (интерактивный), меню.
- **Длинные команды** (tcpdump, strace) нужно ограничивать по времени.
- **Большие файлы:** вывод логов на гигабайты лучше фильтровать grep/tail на сервере.
- **Структурированные данные:** MCP-интеграция надёжнее для API-запросов (например, Proxmox VM list через MCP быстрее и точнее, чем SSH + pvesh).
