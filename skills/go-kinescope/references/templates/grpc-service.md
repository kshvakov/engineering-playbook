## gRPC service (Kinescope template)

Reference: “gRPC Service” template (protobuf + `protoc-gen-go-tags` + validate) and strict layering service/module.

### Expected layout

- `cmd/grpcservice/main.go` — entrypoint, logging, `application_info`, CLI (`urfave/cli`)
- `cmd/grpcservice/action/action.go` — flags+env, vault, build `config`, monitoring server, `service.New()`/`Run()`
- `config/config.go` — typed config (`GRPCAddr`, `Postgres`, optionally Redis/Memcached)
- `internal/service/` — gRPC boundary methods, validation, error translation
- `internal/module/` — business logic & DB (SQL in `queries.go`)
- `proto/` — `proto/<svc>/` sources + `proto/gen/` generated code
- `client/` — reusable gRPC client (if required)
- `spec.yml`, `Makefile`, `version.go`, `.golangci.yml`

### Boundary rules (gRPC)

- **Proto validation**: `req.Validate()` (generated by `protoc-gen-go-tags`) and wrap with `grpc_errors.Wrap(err)` (see `proto-pattern.md`, `validation-grpc.md`)
- **Errors**: wrap/classify/log once (see `policy.md`, `error-handling.md`)
- **Modules**: no retries/transport logic (see `policy.md`, `module-pattern.md`)

### Checklist

- [ ] proto imports `kinescope/protobuf/tag.proto`, uses `(tag.validator)` and `(tag.uuid)`
- [ ] Makefile supports `make proto` and finds `protoc-gen-go-tags` path via `go list -m`
- [ ] `service.New()` pings deps with timeouts and configures Postgres pool
- [ ] gRPC methods validate input via `req.Validate()`
- [ ] modules contain business logic and DB; caching only here
- [ ] SQL in `queries.go`, formatted per `sql-formatting.md`
