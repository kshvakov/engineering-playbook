# Task Formalizer — руководство пользователя (входная точка)

Этот README — **точка входа**. Основной документ для работы “с нуля” — `PLAYBOOK.md`.

> Примечание: детали форматов, шаблоны артефактов и “справочная часть” лежат в `references/`.

## С чего начать (5 минут)

1) Подготовь задачу в `.md` (можно “как есть”, но лучше по шаблону из `PLAYBOOK.md`).
2) Перетащи файл с задачей (`task.md`) в чат Cursor и напиши короткий запрос, что сделать с задачей (1–2 предложения). Отправь сообщение.
3) Если в `MustHaveInputs` есть блокеры — сначала собери ответы (не переходи к Spec/AC/Test).
4) Когда блокеров нет — сформируй пакет артефактов и раздай его по ролям.

Примеры минимальных запросов:

```text
Сделай только секцию ## Routing и перечисли блокеры (MustHaveInputs).
```

```text
Проверь задачу на пробелы и противоречия. Не предлагай решений — только список проблем и вопросы.
```

```text
Собери список вопросов P0–P2 и что нужно предоставить дальше, чтобы можно было писать Spec/AC и тесты.
```

Дальше — по шагам и с примерами: `PLAYBOOK.md`.

## Что давать во входе (чтобы не было домыслов)

- Текст задачи как есть (контекст, зачем, ограничения).
- Любые артефакты: ссылки, скрины, логи/метрики, примеры ошибок, текущий флоу.
- Если вы уже знаете тип: добавьте `TaskTypeOverride: ...`.
- Если нужен не полный пакет: добавьте `FormalizerMode: ...` (см. ниже).

## Overrides (копируй‑вставляй)

### Override типа

```text
TaskTypeOverride: <UX|Perf|Incident|Migration|SecurityAuthZ|SecurityKeys|Integration>
```

### Ограничение объёма

```text
FormalizerMode: <full|questions_only|routing_only>
```

## Playbook (как “вести задачу” до релиза)

Пошаговый процесс, примеры промптов/результатов, ветки под роли, чеклист приёмки и шаблон release notes:

- `PLAYBOOK.md`
- `real-cases.md` — реальные кейсы и переиспользуемые промпты (копилка для будущих обновлений playbook)

Дополнительно: в результате `task-formalizer` теперь есть **переписанное резюме задачи** (user-story, строго по фактам). Его удобно вставлять в тикет как понятную постановку.
